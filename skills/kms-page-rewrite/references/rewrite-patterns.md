# Rewrite Patterns

Use the lightest structure that makes the page easier to decide from, maintain, and reuse.

## Decision / Strategy Page

Use when the page compares options or recommends a direction.

Recommended sections:

1. Conclusion first
2. Context and goal
3. Options comparison
4. Recommended path
5. Risks and dependencies
6. Action list
7. Appendix / details

Good tables:

| Field | Purpose |
|---|---|
| Option | Short option name |
| Suitable when | Trigger condition |
| Pros | Main benefits |
| Cons / risks | Real tradeoffs |
| Required resources | People, data, system, time |
| Final status | Recommended / backup / not recommended |

## Discussion Draft To Decision Page

Use when the source page lists background and options, but the user wants a clearer recommendation.

Rewrite pattern:

1. Rename the page from a discussion label to a decision or recommendation label when appropriate.
2. Put the recommended option and its boundary conditions first.
3. Keep non-recommended options visible as backup or rejected options.
4. Convert option paragraphs into a comparison table with consistent dimensions.
5. Add next actions so the page can move from discussion to execution.

Do not invent numeric thresholds, owners, deadlines, or resource estimates when the source content does not provide them. Put those items in open questions.

## Operational Handover Page

Use when the page tells others how to run or maintain something.

Recommended sections:

1. Current status
2. Scope and non-scope
3. Daily / weekly workflow
4. Exception handling
5. Owner and escalation
6. Known risks
7. Change log

## Project Progress Page

Use when the page tracks ongoing work.

Recommended sections:

1. One-line status
2. Milestones
3. Current blockers
4. Key decisions
5. Next actions
6. Risks
7. Links / appendix

## Quality Rules

- Put the conclusion before detailed evidence.
- Avoid mixing historical notes with current decisions.
- Convert long prose lists into tables when readers need comparison.
- Keep metric names and units consistent across sections.
- Show owner, deadline, and next action when the page is operational.
- If the content has unresolved disagreement, preserve it as a decision point.
